creating migration code:

dotnet ef migrations add Initial --no-build --context DesicContext --project ./Desic.EntityFrameworkCore.Sqlite
dotnet ef migrations add Initial --no-build --context DesicContext --project ./Desic.EntityFrameworkCore.SqlServer


applying migrations to a database:

dotnet ef database update --no-build --connection "Data Source=./desic.db;" --context DesicContext --project ./Desic.EntityFrameworkCore.Sqlite
dotnet ef database update --no-build --connection "Server=(localdb)\MSSQLLocalDB;Database=Desic;Trusted_Connection=True;" --context DesicContext --project ./Desic.EntityFrameworkCore.SqlServer

reverting database to <prior-migration-name> (use 0 for beginning):

dotnet ef database update <prior-migration-name> --no-build --connection "Server=(localdb)\MSSQLLocalDB;Database=Desic;Trusted_Connection=True;" --context DesicContext --project ./Desic.EntityFrameworkCore.SqlServer
dotnet ef database update 0 --no-build --connection "Server=(localdb)\MSSQLLocalDB;Database=Desic;Trusted_Connection=True;" --context DesicContext --project ./Desic.EntityFrameworkCore.SqlServer




creating a bundle executable for migrations (does not apply them to any databases):
(may need to add .csproj to the end of the project relative path)
dotnet ef migrations bundle --no-build --output efbundle --self-contained --verbose --configuration Release --project ./Desic.EntityFrameworkCore.SqlServer --context DesicContext
